## What is wyrmgaze?

**wyrmgaze** is a pentest action graph generator. You describe your attack chain as a series of actions вҖ” each with inputs and results вҖ” and wyrmgaze renders them as visual SVG graphs in three orientations, plus a markdown report. Shared findings are color-coded and connected automatically across actions, making lateral movement and pivot paths instantly visible.

---

## Installation

No dependencies. Pure Python 3.

```bash
git clone https://github.com/youruser/wyrmgaze
cd wyrmgaze
python wyrmgaze.py actions.txt
```

---

## Action file format

One action per line. Wrap the **action** in `*asterisks*` вҖ” everything before it is input, everything after is a result. Multiple inputs and results are separated by `|`.

```
input | *action* | result
input1 | input2 | *action* | result1 | result2
```

Lines starting with `#` are comments and are ignored. Trailing `|` is also ignored.

### Example `actions.txt`

```
target | *nmap* | port 80 | port 135 | port 445
port 80 | *login* | guest
guest | *found* | file1 | user1
file1 | *read* | user2 | user3 | hashes | MD5
MD5 | *crack* | pass1
pass1 | hashes | *decrypt* | pass1 | pass2
pass1 | pass2 | pass3 | user1 | user2 | user3 | *rid-brute-force* | valid_domain_users1
valid_domain_users1 | pass1 | pass2 | pass3 | *spraying* | valid_creds1
valid_creds1 | *creds_vs_services* | SMB1 | RPC1
SMB1 | *rid-brute-force* | valid_domain_users2
valid_domain_users2 | pass1 | pass2 | pass3 | *spraying* | WINRM1
```

## Usage

```bash
python wyrmgaze.py actions.txt
```

Every run creates a folder named after your input file and places all outputs inside:

```
actions/
в”ңв”Җв”Җ actions_horizontal.svg   вҶҗ left-to-right layout
в”ңв”Җв”Җ actions_vertical.svg     вҶҗ top-to-bottom layout
в”ңв”Җв”Җ actions_hybrid.svg       вҶҗ wrapping strips layout
в””в”Җв”Җ actions.md               вҶҗ markdown report (all graphs + table)
```

### Read from stdin

```bash
cat actions.txt | python wyrmgaze.py -
echo "target | *nmap* | port 80" | python wyrmgaze.py -
```

### Dragon color

Pick the banner color with `--color`:

```bash
python wyrmgaze.py actions.txt --color red       # default
python wyrmgaze.py actions.txt --color cyan
python wyrmgaze.py actions.txt --color magenta
python wyrmgaze.py actions.txt --color green
python wyrmgaze.py actions.txt --color blue
python wyrmgaze.py actions.txt --color white
```

The dragon and the `wyrmgaze` logo both render in your chosen color. The eyes always burn yellow.

## Graph layouts

### Horizontal
Each action is a **vertical column** вҖ” input at the top, action ellipse in the middle, results at the bottom. Actions flow left to right. Best for short chains (up to ~8 actions).

```
  [input]        [input]        [input]
     |               |              |
  (action)       (action)       (action)
     |               |              |
  [result]       [result]       [result]
```

### Vertical
Each action is a **horizontal row** вҖ” inputs on the left, action ellipse in the center, results on the right. Actions flow top to bottom. Best for long chains that would be too wide horizontally.

```
  [input]  вҶ’  (action)  вҶ’  [result]
  [input]  вҶ’  (action)  вҶ’  [result]
                            [result]
```

### Hybrid
Actions flow horizontally in strips that wrap vertically when they exceed 1400px. Cross-strip shared values are connected through the inter-strip gap. Best for large action files.

---

## Color coding

Every unique value (finding, credential, service, etc.) gets its own persistent color. If `guest` appears as a result in action 2 and an input in action 3, both boxes share the same color вҖ” making chains instantly traceable at a glance.

| Connection type | Visual |
|---|---|
| Adjacent shared values | Dashed line at node level |
| Non-adjacent shared values | Dashed line routed above (horizontal) or left (vertical/hybrid) |

---

## Markdown output

Every run generates a single `<stem>.md` file alongside the SVGs. It embeds all three graph orientations as image references and includes the full actions table.

The markdown works in:
- **Obsidian** вҖ” native SVG embed with built-in pan and zoom
- **GitHub Pages** вҖ” plain image embed (place `.md` and `.svg` files in the same directory)

## Tips

- Action names can be anything вҖ” `nmap`, `login`, `crack`, `pivot`, `spray`, `exploit`, `custom-script`
- The `*action*` marker can appear anywhere in the line вҖ” inputs are everything before, results are everything after
- Values that appear in multiple actions are automatically color-matched across the entire graph
- The `--color` flag changes the terminal banner only вҖ” graph colors are always auto-assigned
- Run wyrmgaze multiple times on the same folder вҖ” it overwrites existing files cleanly
